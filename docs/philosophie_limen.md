# Philosophie de Limen

Le mot **Limen** désigne un seuil, une frontière, un point de bascule.

Dans ce projet, ce seuil représente le moment où un système d'IA cesse d'agir comme s'il maîtrisait la situation et reconnaît qu'il devrait probablement demander de l'aide.

## Différence avec les approches multi-agents classiques

Les systèmes multi-agents classiques se concentrent sur la collaboration elle-même.

Limen se concentre d'abord sur la **décision de collaborer**.

La question fondatrice du projet est donc :

> Comment un système d'IA peut-il estimer qu'il atteint la limite de sa compétence ?

## Humilité algorithmique

Limen cherche à introduire une forme d'humilité algorithmique :

- ne pas répondre seul quand le risque est trop élevé ;
- consulter lorsque l'incertitude devient significative ;
- accepter la délégation lorsque le domaine dépasse la compétence estimée.

## Pourquoi la mémoire partagée est essentielle

Multiplier les agents ne suffit pas. Sans mémoire commune, les agents répètent les mêmes coûts de contexte et reconstruisent sans cesse l'état du projet.

Limen considère donc la mémoire comme une composante cognitive à part entière, et pas comme un simple cache technique.
